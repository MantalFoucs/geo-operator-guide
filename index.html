内容运营使用说明

适用于 GEO Skills 的首次安装和日常使用。日常只需使用 Codex 和企微智慧表格。

首次安装

已经可以正常使用的同事跳过这一节。首次安装只做一次；完成后按下面的日常流程操作。

点击展开：安装 GEO Skills
1

安装 Git

在 Windows 搜索并打开 PowerShell，执行：

winget install --id Git.Git -e --source winget

完成后关闭 PowerShell，再重新打开一个普通 PowerShell 窗口。

打开“终端”，执行：

xcode-select --install

按系统提示完成安装。已安装开发者工具时，这一步可跳过。

2

下载 Skill 仓库

git clone https://github.com/MantalFoucs/GEO-skills.git "$env:USERPROFILE\.codex\GEO-skills"
git clone https://github.com/MantalFoucs/GEO-skills.git "$HOME/.codex/GEO-skills"
3

安装两个 Skill

以下命令按顺序逐行执行：

New-Item -ItemType Directory -Force "$env:USERPROFILE\.codex\skills"
cmd /c mklink /J "%USERPROFILE%\.codex\skills\geo-topic-generation" "%USERPROFILE%\.codex\GEO-skills\geo-topic-generation"
cmd /c mklink /J "%USERPROFILE%\.codex\skills\geo-article-production" "%USERPROFILE%\.codex\GEO-skills\geo-article-production"
mkdir -p "$HOME/.codex/skills"
ln -s "$HOME/.codex/GEO-skills/geo-topic-generation" "$HOME/.codex/skills/geo-topic-generation"
ln -s "$HOME/.codex/GEO-skills/geo-article-production" "$HOME/.codex/skills/geo-article-production"
4

配置企微自动写入

需要自动创建企微文档、写入选题库和内容库时，向陈乙辉索取配置内容。不要在 Codex 对话、工作群或 GitHub 中发送 Key。

New-Item -ItemType Directory -Force "$env:USERPROFILE\.config\geo-skills"
notepad "$env:USERPROFILE\.config\geo-skills\.env"
mkdir -p "$HOME/.config/geo-skills"
touch "$HOME/.config/geo-skills/.env" && open -e "$HOME/.config/geo-skills/.env"

打开后粘贴陈乙辉提供的配置内容，保存即可。

5

重新打开 Codex

完全退出并重新打开 Codex，然后新建“选题”和“文章”两个对话开始使用。

点击展开：GitHub 仓库没有权限怎么办

把 GitHub 用户名发给陈乙辉，接受他发出的私有仓库邀请后，再重新执行第 2 步下载命令。

在 Codex 中新建三个对话

选题、文章和更新分别放在三个对话中。后续继续在对应对话里输入要求，不要把不同任务混在一起。

会话一：GEO 选题库

只负责生成选题、避开重复题、写入企微选题库。

请使用 geo-topic-generation Skill。这个会话只负责生成、去重和写入 GEO 选题。

会话二：GEO 内容生成

只负责生成母版、生成平台版本、写入企微内容库。

请使用 geo-article-production Skill。这个会话只负责根据已审核选题生成文章并写入企微。

会话三：GEO Skill 更新

只负责更新本机的选题和文章 Skill，不生成任何内容。

你是 GEO Skills 更新助手。这个对话只负责更新本机的 GEO Skills。收到“更新 GEO Skills”后，执行更新、检查结果，并提醒我重新打开 Codex。

日常流程

1. 生成选题在 Codex 产出候选题
2. 审核选题在企微决定要不要写
3. 生成母版审核并修改文章正文
4. 生成平台版本按勾选平台轻适配
文章会自动新建企微文档，并在内容库新增记录。无需复制全文到表格，也无需手工填写编号、创建时间和文档 docid。

一、生成和审核选题

在 Codex 怎么做

打开GEO 选题库会话，描述这次要解决的人群和问题。

生成 10 个官方常青选题，并写入企微选题库。 方向：本地生活社群团购。 面向：有社区群或线下门店客户的团长。 要求：优先解决开团、选品、履约和售后中的具体问题，避开已有选题。

在企微表格怎么做

打开选题库，查看标题、推荐理由和目标。保留真正值得写的题目，后续从这一行复制标题和目标即可。

示例
标题：社群团长做本地生活接龙时，哪些商品根本不适合拿来开团？
目标：明确高投诉、高履约风险、低复购商品的风险。

二、根据选题生成母版文章

1

从选题库复制标题和目标

在企微选题库确认要写后，复制该行的标题和目标。标题决定文章要回答的问题，目标决定文章写完后读者应获得什么。

2

在 GEO 内容生成会话发送任务

请使用 geo-article-production Skill，生成一篇母版文章并写入企微文档和内容库。 标题：社群团长做本地生活接龙时，哪些商品根本不适合拿来开团？ 目标：明确高投诉、高履约风险、低复购商品的风险，减少售后和信任损耗。 补充要求：面向新团长，保留具体场景举例和开团前判断标准。
3

在内容库确认母版

内容库会出现一条类型：母版的记录。打开“文章正文”链接审核正文；需要改内容时，直接在企微文档里修改。确认可以作为其他平台的基础稿后，把该行审核状态改为母版已确认。

母版是什么？
母版是已经调好结构、观点、场景和 FAQ 的原稿。之后生成其他平台时，会以它为基础轻适配，不会重新写成另一篇完全不同的文章。

三、生成需要的平台版本

1

在母版行勾选待生成平台

在内容库找到状态为母版已确认的母版，在待生成平台勾选这次需要生成的平台。比如需要公众号和知乎，就勾选微信公众号和知乎。不要填写分发平台。

2

复制母版编号

编号形如GEO-20260714-163240-4840。它是母版的唯一识别码，用来让 Codex准确读取对应正文。

3

在 GEO 内容生成会话发送任务

请使用 geo-article-production Skill，根据母版编号生成已勾选的平台版本。 母版编号：GEO-20260714-163240-4840 要求： 1. 读取母版企微文档正文； 2. 只生成待生成平台中已勾选的平台； 3. 保留母版的核心观点、章节、示例和 FAQ，只做轻适配； 4. 每个平台版本新建企微文档并写入内容库； 5. 自动回填编号、创建时间和企微文档 docid； 6. 回写母版的平台生成状态； 7. 不修改分发平台字段。
4

查看新生成的两条记录

内容库会新增两条记录，类型会直接标识平台，便于区分和筛选。

类型标题变化保留不变的部分
平台版本：微信公众号更适合深度阅读的标题和开头母版的观点、案例、判断标准、FAQ
平台版本：知乎疑问句标题，开头先给结论母版的观点、案例、判断标准、FAQ
平台版本的编号会继承母版编号，例如：GEO-20260714-163240-4840-WECHAT、GEO-20260714-163240-4840-ZHIHU。母版的待生成平台会清空，平台生成状态会变为“已生成”。

一个完整例子

假设运营决定写“如何做好社群运营”。流程不是重新复制文章去改，而是：

  1. 在内容生成会话生成如何做好社群运营的母版。
  2. 在企微文档中把案例、表达和 FAQ 调整到满意，再把母版状态改为母版已确认。
  3. 在母版行勾选微信公众号和知乎。
  4. 复制母版编号，发送平台版本任务。
  5. 内容库得到平台版本：微信公众号和平台版本：知乎两条独立记录，各自有文章链接、编号和 docid。

质量不满意时，怎样调整指令

不要只说“写得不好”“再丰富一点”。先指出哪里要保留、哪里要改、改完要达到什么效果。继续在原来处理选题或文章的对话里发送补充要求，不需要新建会话。

生成选题：把范围收窄

选题太像、太泛或不适合当前内容计划时，说明人群、场景、要解决的问题和不要出现的角度。

请使用 geo-topic-generation Skill，重新生成 10 个选题并写入企微选题库。 本次只面向：刚开始做社区团购的新团长。 只写场景：第一次开团、选品判断、订单和售后处理。 不要出现：群接龙和微信群文字接龙的区别、泛泛的“社群团购怎么做”。 标题要求：每个标题只解决一个具体问题，避免与已有选题语义重复。

选题调整的判断方式

把“太泛”换成具体边界。

不够具体
再生成一些更好的社群运营选题。

更有效
只写社区团长在开团前容易判断错的事，每个标题要能直接带出检查标准或操作步骤。

生成母版：明确要补什么

文章缺例子、内容空、结构不顺时，直接把补充要求写在标题和目标后面。

请使用 geo-article-production Skill，按以下要求重写母版文章并写入企微。 标题：社群团长做本地生活接龙时，哪些商品根本不适合拿来开团？ 目标：明确高投诉、高履约风险、低复购商品的风险。 修改要求： 1. 开头先给明确结论，不要先解释概念； 2. 每个主要判断点都要有一个典型场景，写清“发生什么、为什么有风险、团长该怎么判断”； 3. 保留开团前检查标准； 4. 删除空泛的“提升效率、建立信任”等表达； 5. 不编造真实商家、订单量、专家观点或数据。

文章调整的判断方式

以下四类要求最容易让文章质量发生明确变化：

  • 读者：面向谁，例如新团长、实体店主、宝妈团长。
  • 结构：先给结论、需要几个判断点、是否保留 FAQ。
  • 内容：需要案例、清单、操作步骤，还是更深的分析。
  • 边界：哪些词、观点、数据或表达不能出现。

平台版本：只改适配方式，不推翻母版

平台稿与母版差别过大时，要在任务里明确“轻适配”。

请使用 geo-article-production Skill，根据母版编号生成已勾选的平台版本。 母版编号：GEO-20260714-163240-4840 补充要求：保留母版的论点、章节顺序、典型场景、判断标准和 FAQ；只调整标题、开头、关键词表达和语气。不要写成另一篇新文章。

改哪里最合适

只改错别字、句子或排版：直接在对应企微文档改，不必重新生成。

要改观点、案例或文章结构：先修改并确认母版，再重新勾选需要的平台生成版本。把旧的平台版本审核状态改为不通过，新版本会以待审核出现，避免混淆。

一次只调整最重要的 2 至 5 条要求。要求越具体，结果越稳定；同一条任务里同时要求“更短、更详细、更活泼、更专业”会让结果失去重点。

更新 Skill

使用上一步创建的GEO Skill 更新对话。这个对话只用来更新，不用来生成选题或文章。

你是 GEO Skills 更新助手。这个对话只负责更新本机的 GEO Skills。 当我说“更新 GEO Skills”时： 1. 在本机执行 git pull，更新路径为 $env:USERPROFILE\.codex\GEO-skills$HOME/.codex/GEO-skills； 2. 检查并告诉我更新是否成功、更新了什么； 3. 不修改企微表格、文章、Key 或配置文件； 4. 更新成功后，提醒我完全退出并重新打开 Codex。

以后收到更新通知，只需要打开这个对话，发送：

更新 GEO Skills

更新完成后，需要手动完全退出并重新打开 Codex，新的 Skill 规则才会被重新加载。

遇到问题时

现象处理方式
生成了正文，但企微没有新增记录把 Codex 返回的完整错误和操作时间发给陈乙辉，不要自行补填编号或 docid。
平台版本没有生成检查母版是否为母版已确认，是否勾选了待生成平台，以及发送的母版编号是否正确。
Codex 找不到 Skill先完全退出并重新打开 Codex；仍无效时执行“更新 Skill”的两条命令，并把报错发给陈乙辉。
Key、GitHub 权限或企微写入问题直接找陈乙辉处理，不要自行在聊天记录、文档或 GitHub 中粘贴 Key。
文章里有事实拿不准在企微文档中标出问题，不要自行补充产品功能、数据或外部案例。
